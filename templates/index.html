Smart Matching

Finds the nearest compatible donor using the hospital's exact coordinates and blood-type compatibility rules — not just an exact type match.

Pub/Sub Alerts

Every match publishes to a GCP Pub/Sub topic. Subscribers send SMS/email to the donor and ping the admin dashboard simultaneously.

Secure Login

Push or pull delivery

Donor notification Push

GCP calls /pubsub/push on your Cloud Run service. Your handler reads the message and sends an SMS/email to the matched donor.

POST /pubsub/push { "event": "BLOOD_REQUEST_MATCHED", "donor_name": "Rahul Naik", "donor_phone": "94221xxxxx", "hospital_name": "GMC Panaji", "blood_type": "O+", "distance_km": 3.2 }

Admin dashboard Push

A second subscription delivers every event to the admin panel endpoint, updating request status in real time without polling the database.

POST /pubsub/push { "event": "BLOOD_REQUEST_NO_MATCH", "hospital_name": "Asilo Mapusa", "blood_type": "AB-", "urgency": "CRITICAL" }
